🔄 HOW IT WORKS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. User uploads JSON file with credentials
       ↓
  2. Frontend generates commitments (keccak256 hashes)
       ↓
  3. Frontend builds Merkle tree (sorted-pair hashing)
       ↓
  4. Frontend calls API: POST /api/batch/submit
       ↓
  5. API publishes to blockchain (2 transactions):
       • addCommitmentsBulk(commitments[])
       • submitBatch(merkleRoot)
       ↓
  6. API returns: batchId, merkleRoot, txHash, proofs
       ↓
  7. Frontend stores in localStorage
       ↓
  8. Frontend displays success + Polygonscan link
